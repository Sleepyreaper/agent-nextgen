💰 COST & SPEND

1. **CI/CD Waste — Staging deploys on every branch push** (#71)
   The GitHub Actions workflow deploys to staging on every push to `feature/**`, `fix/**`, `dev/**` — which includes all sweep branches. With 3 automated agents pushing daily, that's 3+ unnecessary staging deployments per day, burning ~150-300 Actions minutes/month. Recommendation: exclude sweep branches from the trigger.

2. **Unbounded telemetry/audit tables — no retention** (#69)
   Six tables (`telemetry_events`, `agent_audit_logs`, `agent_interactions`, `file_upload_audit`, `training_feedback`, `user_feedback`) grow without bound. At moderate usage, expect 1-5 GB/month at ~$0.115/GB. Recommendation: 90-day retention with archive to cool blob storage.

3. **Azure OpenAI SKU is S0** — this is the standard pay-as-you-go tier, which is appropriate. No overspend concern here.

4. **Log Analytics retention is 30 days** — reasonable. No action needed.

5. **4-Tier Model Architecture** — well-structured with premium/merlin/workhorse/lightweight tiers. The tiering looks intentional and optimized. Models are configured via Key Vault, which is the right pattern.

🏥 HEALTH STATUS

1. **Gunicorn timeout conflict** (#68)
   `startup.py` hardcodes 600s timeout while `gunicorn.conf.py` defaults to 120s. Depending on which startup path is used, a stuck request could hold a worker for 10 minutes. Recommendation: remove hardcoded values from `startup.py`, use env vars through `gunicorn.conf.py` as single source of truth.

2. **app.py is 8,933 lines / 390KB** — previously flagged in #23, now grown from 8,310 to 8,933 lines (623 lines added). Still a critical maintainability risk. Willie's latest sweep added 35 more lines. This file needs to be broken up.

3. **Background event loop pattern** — the shared `_bg_loop` in app.py (Issue #52 fix) is sound architecture. Properly handles async-in-sync Flask. No concerns.

4. **Error handling** — agent imports use try/except with graceful fallback (`ArielQAAgent = None`). Good pattern.

5. **Health check in Dockerfile** — present and correct (`curl -f http://localhost:8000/`). ✅
