## Detailed Findings

### 🔴 #1: Hardcoded Credentials in Git History (CRITICAL)

**Commit:** `d27ed6e` (deleted file `testing/run_milo_validation.py`)  
**Credentials:** `***REMOVED***` / `***REMOVED***`  
**Target:** `https://nextgen-agents-web.azurewebsites.net`

This is the skeleton key. Even though the file was deleted, git remembers everything. Anyone who has ever cloned this repo has these credentials. If the password hasn't been rotated, game over.

**Action required:** Rotate password NOW, then rewrite git history with BFG or `git filter-repo`.

### 🟠 #2: WAF Bypass — Direct Backend Access (HIGH)

The App Service has no IP access restrictions. The `azure-deploy.json` ARM template defines no `ipSecurityRestrictions`. The app doesn't validate `X-Azure-FDID` headers. Your expensive Azure Front Door Premium WAF? It's a suggestion, not a gate.

### 🟠 #3: Prompt Injection via Student Documents (HIGH)

Every AI agent concatenates raw extracted document text into LLM prompts. A student (or attacker) can embed instructions in a PDF that manipulate scores, leak data from agent context, or contaminate the evaluation pipeline. This is the nature of LLM applications, but zero mitigation is in place.

### 🟡 #4-6: Medium Findings

See individual GitHub issues for details. The CSP `unsafe-inline` is a known technical debt item (14 templates with inline handlers). The debug endpoint is a data exfiltration shortcut. The resource name exposure aids reconnaissance.
