💰 COST & SPEND

1. **Log Analytics PerGB2018 pricing** (#92) — Azure deploy template uses pay-as-you-go logging at ~$2.76/GB ingested. With full OpenTelemetry instrumentation (Flask, requests, psycopg, per-agent token telemetry), ingestion could be 1-5 GB/day ($83-$414/month). Recommend reviewing actual usage in Portal and considering sampling.

2. **Gunicorn timeout inconsistency** (#87) — `gunicorn.conf.py` says 120s, `.webappconfig.json` says 600s. The real value is 120s. Long-running AI orchestration (Smee pipeline) could exceed this, causing worker kills and wasted Azure compute on restarts.

3. **Gaston cleanup runs every boot** (#90) — Startup migration scans the entire `applications` table for Gaston data on every worker start. Once cleaned, this is pure waste — unnecessary DB I/O on every deploy/restart.

4. **Model tier architecture is well-designed** ✅ — The 4-tier model system (Premium → Merlin → Workhorse → Lightweight) with dedicated Vision tier is a solid cost optimization. Agent-to-tier mapping looks appropriate. No obvious cases of expensive models being used where cheap ones would work.

🏥 HEALTH STATUS

5. **app.py is 17KB (down from 300KB+)** ✅ — The Blueprint refactor (Issue #23) was a massive improvement. `app.py` is now a clean entry point with proper security headers, CSP, and error handling.

6. **CSP policy is strong but incomplete** (#91) — `script-src` and `script-src-elem` are nonce-based (excellent). However, `script-src-attr: 'unsafe-inline'` persists due to 40+ templates with inline event handlers. This is acknowledged tech debt.

7. **Database connection pooling is properly configured** ✅ — `psycopg_pool.ConnectionPool` with `min_size=2, max_size=10`, lazy initialization, and proper `getconn/putconn` lifecycle. Falls back gracefully to single connection.

8. **Retention scheduler runs in-process** (#88) — `threading.Timer` in gunicorn workers risks duplicate runs (2 workers = 2 schedulers) and missed cleanups if workers recycle. File-based lock won't work on ephemeral Azure filesystems.

9. **Error handling is comprehensive** ✅ — Custom 404/500/429 handlers, no framework detail leakage, proper logging with exc_info.
